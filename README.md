# Action Architect

Action Architect is a full-stack productivity platform built from three coordinated applications:

- Backend API: `src/index.ts` launches the Express API and Socket.io server.
- Worker: `src/workers/aiWorker.ts` processes background BullMQ AI jobs.
- Frontend: `src/start.ts` is the TanStack Start application entry, with Vite serving the browser UI.
- SSR server entry: `src/server.ts` is used by TanStack Start for server-side rendering and error handling.

This repository contains local dev tooling, build scripts, and documentation for running and debugging each part.

---

## Quickstart

1. Install dependencies:

```bash
npm install
```

2. Create a `.env` file from the template:

```bash
cp .env.example .env
```

Then update the values in `.env`:

- `NODE_ENV` (development | staging | production)
- `BACKEND_PORT` (default `3000`)
- `DATABASE_URL` (Postgres connection string - REQUIRED)
- `JWT_SECRET` (REQUIRED - minimum 32 characters in production)
- `LLM_API_KEY` (REQUIRED - for AI plan generation)
- `REDIS_HOST` (default `localhost`)
- `REDIS_PORT` (default `6379`)
- `REDIS_PASSWORD` (optional)
- `WS_BASE_URL` (default `http://localhost:3000`)
- `API_BASE_URL` (default `http://localhost:3000`)
- `CORS_ORIGIN` (default `http://localhost:5173`)
- `LLM_MODEL` (default `deepseek-v4-pro`)
- `LLM_FALLBACK_MODEL` (default `deepseek-v4-flash`)
- `GUEST_USER_EMAIL` (default `guest@action-architect.local`)
- `GUEST_USER_NAME` (default `Guest User`)
- `GUEST_USER_PASSWORD` (default `guest-password`)

3. Start Postgres and Redis (recommended):

```bash
docker compose up -d db redis
```

4. Generate Prisma client and run migrations:

```bash
npx prisma generate
npx prisma migrate dev --name init
```

5. Run the project:

- Start all services together in one terminal:

```bash
npm run dev
```

- Or run each service individually in its own terminal:

Backend API:

```bash
npm run dev:backend
```

Worker:

```bash
npm run dev:worker
```

Frontend:

```bash
npm run dev:frontend
```

## Application entry points

**Backend:**

- API bootstrap: `src/index.ts` (HTTP server + Socket.io)
- Express app setup: `src/app.ts` (middleware, routes)
- SSR server entry: `src/server.ts` (TanStack Start SSR handler)
- Worker entry: `src/workers/aiWorker.ts` (BullMQ background jobs)
- Queue setup: `src/queues/aiQueue.ts` (job producer)

**Frontend:**

- Application entry: `src/start.ts` (TanStack Start bootstrap)
- Router configuration: `src/router.tsx` (TanStack Router factory)
- Generated route tree: `src/routeTree.gen.ts` (auto-generated by TanStack Router)
- Frontend routes: `src/routes/index.tsx`, `src/routes/__root.tsx` (page definitions)

## Build and production

Build both frontend and backend:

```bash
npm run build
```

Build frontend assets and SSR artifacts:

```bash
npm run build:frontend
```

Compile backend sources into `dist/`:

```bash
npm run build:backend
```

Run compiled production backend:

```bash
npm start
```

## Run scripts

- `npm run dev` — start backend API, worker, and frontend together using `concurrently`
- `npm run dev:backend` — run the backend API with `ts-node-dev` from `src/index.ts`
- `npm run dev:worker` — run the BullMQ worker from `src/workers/aiWorker.ts`
- `npm run dev:frontend` — run the Vite frontend development server
- `npm run build` — full production build (frontend + backend)
- `npm run build:frontend` — build frontend and SSR artifacts
- `npm run build:backend` — compile backend TypeScript sources
- `npm start` — run compiled backend from `dist/index.js`
- `npm run lint` — run ESLint across `src/**/*.ts` and `src/**/*.tsx`

## Debugging

The workspace includes VS Code launch configurations in `.vscode/launch.json`:

**Individual services:**

- `🚀 Run: Frontend (npm)` — start Vite frontend dev server
- `🚀 Launch: Backend (API)` — run backend API with Node inspector (port 9229)
- `🚀 Launch: Worker (BullMQ)` — run worker with Node inspector (port 9230)
- `🖥️ Debug: Frontend (Edge)` — attach Edge browser debugger to frontend (localhost:8080)

**Compound configurations (run multiple services at once):**

- `🌟 Full Stack (All Services)` — backend, worker, and frontend
- `🎯 Backend + Worker (No Frontend)` — backend and worker only
- `💻 Frontend + Backend (API)` — frontend and backend API
- `🖥️ Frontend + Edge Debugger` — frontend with browser debugging
- `⚙️ Backend + Worker + Edge Debugger` — backend, worker, and browser debug
- `🧪 Full Stack + Edge Debugger (All + Browser Debug)` — all services with browser debugging

Select a configuration in the VS Code Run and Debug panel (Ctrl+Shift+D) and press Play, or run services manually from the terminal using the npm scripts listed above.

## Selected API endpoints

**Authentication:**

- `POST /api/auth/register` — create account
- `POST /api/auth/login` — authenticate user
- `GET /api/auth/me` — get current user

**Plans:**

- `GET /api/plans` — list user plans
- `POST /api/plans/generate` — AI-generate plan from input
- `GET /api/plans/:planId` — get single plan with steps
- `GET /api/plans/:planId/progress` — get plan completion metrics

**Steps:**

- `PUT /api/steps/:stepId` — update step
- `PATCH /api/steps/:stepId/complete` — mark step complete
- `POST /api/steps/:stepId/time` — log hours on step

**Time tracking:**

- `DELETE /api/time-entries/:entryId` — delete time entry

**Real-time chat:**

- Socket.io connection at `/ws/chat` — live chat on steps with AI

## Troubleshooting

- If LLM calls fail, verify `LLM_API_KEY` and `LLM_MODEL` / `LLM_MODEL_FALLBACK`.
- If port `3000` is in use, set `BACKEND_PORT` in `.env` or run `BACKEND_PORT=3001 npm run dev`.
- Use `npm run lint` to validate the TypeScript and React codebase.
